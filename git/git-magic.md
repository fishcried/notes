#  git黑魔法 

## 运算符号

**`^`**

父

- `A^`

**`^n`**
第几个父祖先

**`~n`**
表示连续n个`^`

**一个版本**

包含该版本的所有历史

**两个或多个版本**

相当于单个版本使用时的并集

**`^A`**

否定,去除这个版本极其历史版本


**`G..D`**

两个版本之间,相当于^G D


**`G...D`**

三个点表示去除两个版本功能能够访问的除外

**`B^@`表示除了本次B自身提交的历史提交**

**`B^!`表示不包含历史的提交**

# 写好commit

- 第一行好简短.
- 第二行永远是空行
- commit要说明"为什么修改是必要的","如何解决问题","可能影响那些地方"

如果有特殊要求的话,可以使用模板.
